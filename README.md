# Food Delivery Platform
## Introduction
This repository contains a Java-based food delivery platform developed using Java Swing and modeled after the Uber Eats application. It follows an object-oriented programming approach to build an interactive graphical user interface and handle user interactions.<br><br>
Due to time constraints, the system is not fully completed; however, it demonstrates core concepts such as GUI development, object-oriented design, database integration, and the implementation of a basic food ordering workflow.<br><br>
Source Code: https://github.com/wilson30139/Java-DB-Food-Delivery-Platform/tree/main/src

## Feature
- Browse restaurants and menu items
- Place food orders and create new orders
- Play background music during system usage
- Place and manage orders
- Java Swing-based graphical user interface

This project is divided into five tabs: **Home, Search, List, Account, and Breakout Clone**, as shown in Figure 1. I was mainly responsible for developing the Home, Search, and the background music feature in the Account section. The List, remaining Account features, and Breakout Clone were developed by two other team members. The following sections briefly introduce the Home and Account functionalities.

Figure 2 displays the available food items in a selected restaurant, including item names and prices. Users can select quantity and size based on their preferences. If they decide not to order a selected item, it can be removed from the cart. After completing the selection, users can place an order, and the order page will display all selected items.

Figure 3 shows the Account Settings page. The top section includes background music controls, allowing users to play, pause, and resume music playback. Below that, "Your Favorite" stores favorite restaurants, "Payment Setting" manages payment information, and "Help" provides assistance. Due to time constraints, these sections were only implemented at the UI level without full backend functionality. Additionally, the four items at the bottom were also only implemented as interfaces without functional logic.
